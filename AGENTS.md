# AGENTS.md (Codex 작업 지침)

## 기본 원칙
- 모든 답변은 한국어로 작성한다.
- 코드를 변경할 때는 반드시 버전 번호를 올린다. (예: v0.3.1 -> v0.3.2)
- 변경점은 CHANGELOG 형식으로 요약한다.
- 코드는 가능한 한 "diff 패치" 형태로 제시한다.
- 프로젝트 구조/기존 동작을 불필요하게 깨지 않는다. 깨질 수 있으면 대안을 2개 제시한다.

## 작업 방식
1) 먼저 문제 원인/요구사항을 3줄로 요약
2) 가능한 해결책 2~3개 비교(장단점)
3) 최선안으로 패치(diff) 제공
4) 테스트 체크리스트 제공
5) 롤백 방법 제공

## 금지/주의
- 임의로 의존성 추가하지 말 것(필요 시 이유와 대안 제시).
- 큰 리팩터링은 단계별 커밋처럼 나눠서 제안할 것.
