# 성주(영주) + 성·마을 경영 시뮬레이션 게임 기획서

> 목표: 플레이어가 **성주(영주)**가 되어 **성 + 마을**을 운영·확장하며  
> **경제(생산/물류) · 사회/정치(정통성/파벌) · 위기(사건/전쟁/재난)**를 균형 있게 관리하는 경영 시뮬레이션을 구현한다.

---

## 1) 게임 목표와 핵심 재미 축

### 핵심 목표
- **정착지 생존/성장**: 인구 유지 → 생산 안정화 → 기술/인프라 확장
- **권력 유지**: 세금/치안/정통성 관리 → 봉신·길드·교회·민중과 균형
- **위기 대응**: 전염병/흉년/약탈/전쟁/정치 사건을 “자원 + 판단”으로 해결

### 플레이 감각(추천 3축)
1) **경제/물류 축**: 생산라인 최적화, 공급망 병목 해결  
2) **사회/정치 축**: 계층·파벌 관리, 정책 선택의 트레이드오프  
3) **사건/서사 축**: 랜덤 이벤트+인물 관계로 반복 플레이 서사 생성

---

## 2) 필수 코어 변수(데이터) 목록

### A. 인구·노동
- `Population(총인구)` / `Households(가구)` / `Workforce(노동가능 인구)`
- `ProfessionCounts(직업별 인구)` : 농부/목수/광부/병사/상인/성직자/의사 등
- `Skill(숙련도)` : 생산량, 품질, 사고율에 영향
- `Health(건강)` / `Hunger(굶주림)` / `Morale(사기·행복)` / `Loyalty(충성)`

### B. 자원·경제
- **기본 자원**: `Food`(곡물/고기/생선), `Wood`, `Stone`, `Iron`, `Tools`
- **화폐/재정**: `Gold(Treasury)` / `Income(수입)` / `Expense(지출)`
- **저장/부패**: `StorageCapacity`, `SpoilageRate`
- **가격/수급**: `MarketPrice(가격)`, `SupplyDemand(수급)`

### C. 생산·건설
- `BuildingLevel(건물 레벨)` / `ConstructionQueue(건설 큐)`
- `ProductionRate(생산량)`, `InputConsumption(투입량)`
- `Maintenance(유지비/수리)` / `Durability(내구도)`

### D. 질서·치안·법
- `Security(치안)` / `Crime(범죄율)` / `Order(질서)`
- `Justice(재판/처벌 강도)` / `Corruption(부패)`

### E. 외부 환경
- `Season(계절)` / `Weather(날씨)` / `Fertility(비옥도)`
- `ThreatLevel(위협)` : 약탈자, 적국, 야수, 반란 위험

### F. 권력(영주 시스템)
- `Legitimacy(정통성/합법성)` : 세금 저항·반란·외교에 직결
- `Reputation(평판)` : 이주/무역/동맹에 영향
- `VassalOpinion(봉신 평판)`, `ClergyRelation(교회)`, `GuildRelation(길드)`

## 3) 코어 시스템(필수 구현 우선순위)

### 3.1 1순위: “생존/경제 루프”
1) **식량 루프**
- 농장/목장/어장 → 저장고 → 소비(인구)  
- 부족 시: 사기↓, 질병↑, 범죄↑, 이주/사망↑

2) **노동 배치**
- 직업 배치(계절성 포함) + 숙련도 성장  
- “농번기 vs 건설 vs 군사”의 우선순위 딜레마 제공

3) **건설/확장**
- 주거 → 생산시설 → 저장/도로 → 성벽/방어  
- 건설은 자원 + 노동 + 시간 + 유지비(중요)

4) **기본 세금/예산**
- 세금(수입) ↔ 행복/충성(불만) 트레이드오프  
- 예산은 건설·급여·군비·구휼·치안에 분배

### 3.2 2순위: “리스크/위기 루프”
1) **사건 시스템**
- 흉년/홍수/전염병/화재/약탈/상단 방문/귀족 방문  
- 사건은 “즉시 선택 + 후속 여파(지속 디버프/버프)” 구조 추천

2) **치안/범죄**
- 치안 낮음 → 도난(재고 감소), 방화/폭동 확률 증가  
- 처벌 강도↑ 시 범죄↓ 대신 행복/정통성↓ 가능

3) **방어/군사**
- 성벽/탑/병영/민병대  
- “군비 유지비 vs 경제 성장” 균형이 핵심 재미

### 3.3 3순위: “성주/정치 루프”
1) **정통성/평판**
- 정책/재판/전쟁/기근 대응이 정통성에 누적 영향  
- 정통성 낮으면: 세금 효율↓, 반란↑, 외교 불리

2) **파벌(민중/길드/교회/봉신)**
- 파벌별 요구/불만/호의 수치 + 임계치 이벤트(파업, 탄원, 성전 요구, 봉신 반발)

---

## 4) 추천 심화 요소(깊이를 만드는 장치)

### A. 물류/공급망 심화
- 운송 단위(운송꾼/마차), 도로 상태에 따른 이동속도
- 병목: 저장고 포화 → 생산 중단, 운송 부족 → 공급 지연
- 품질: 좋은 도구/숙련도 → 생산품 품질↑ → 무역가치↑

### B. 계층/계급 시스템
- 계층: 농노/평민/상인/장인/귀족/성직자  
- 계층별 소비/세금/정치 요구 차등
- 상층 만족 → 세수·투자↑, 하층 불만 → 폭동↑

### C. 인물/가문/관료 드라마
- 핵심 NPC: 집사(재정), 기사단장(군사), 재상(외교), 주교(종교)
- 성향(탐욕/충성/야망) + 관계(호감/원한)
- 인물 기반 사건: 부패/쿠데타/정치 거래 등

### D. 법·정책 트리
- 세금(인두세/지대/관세), 군역, 구휼, 종교 관용, 시장 규제
- 즉시효과 + 장기 누적치(정통성/성장률/불만)로 설계

### E. 기술·문화·교육
- 기술: 윤작, 제철, 건축, 의학, 군사훈련, 행정
- 문화: 학교/도서관/성당/축제
- 장기 버프(숙련 성장↑, 질병↓, 정통성↑ 등)

### F. 외교/무역/전쟁 확장
- 주변 영지/도시국가/수도 관계 수치
- 조약(통상/불가침/혼인), 조공, 용병
- 방어 중심 → 원정/공성으로 확장

## 5) 핵심 상관관계(루프 맵)

### 경제·생존(긍정 루프)
- **식량↑** → 건강↑/사기↑ → 생산성↑ → 세금 기반↑ → 인프라↑ → 식량↑

### 불만·치안(부정 루프)
- **세금↑/식량↓** → 행복↓/충성↓ → 범죄↑/파업↑ → 생산↓/재정↓ → 구휼↓ → 악화

### 정통성(정치 루프)
- **공정한 재판/구휼/승리** → 정통성↑ → 세금 저항↓/이주↑/외교↑  
- **부패/폭정/패전** → 정통성↓ → 반란↑/봉신 이탈↑/무역 불리

### 물류 병목(최적화 루프)
- 생산시설↑만으로는 한계  
- **운송·저장·도로** 부족 → 공급 지연 → 전체 효율↓  
→ “병목 찾기”가 반복 재미가 됨

---

## 6) 구현용 모듈 설계(권장 구조)

### 데이터 레이어
- `Resource`(종류, 부패, 단위, 가치)
- `Building`(레벨, 입력/출력, 인력, 유지비, 내구)
- `PopUnit`(가구/개인, 직업, 숙련, 건강/사기, 계층)
- `Faction`(요구, 호감, 임계 이벤트)
- `Policy`(비용, 조건, 즉시효과, 장기효과)
- `Event`(발동조건, 선택지, 결과, 후속상태)

### 시뮬레이션 틱(예시)
- **일 단위**: 소비/생산/질병 체크  
- **주 단위**: 세금/급여/시장 가격 업데이트  
- **월/계절 단위**: 수확/이주/정치 이벤트 가중치 변경

---

## 7) 개발 로드맵(필수 vs 추천)

### MVP(최소 재미 보장)
- 인구/직업 배치 + 식량/기초자원 생산 + 건설 + 저장 + 세금/예산 + 계절 + 약탈 이벤트 1종

### 코어 완성(중독성 확보)
- 치안/범죄 + 기본 파벌 2개(민중/길드) + 무역 + 질병 이벤트 + 방어(성벽/병사)

### 딥 확장(장기 가치)
- 인물/관료 + 정통성/법·정책 + 계층 + 공급망(운송) + 외교/봉신

---

## 8) 추천 지표 UI(이해/학습 강화)
- 상단 핵심 6개: **인구 / 식량(일수) / 금고 / 행복 / 치안 / 정통성**
- 경고 패널: “식량 12일”, “범죄 급증”, “봉신 반발 임계치 80%”
- 원인 설명: 행복↓ 기여도(세금 40%, 식량 35%, 치안 25%)처럼 분해 표시
